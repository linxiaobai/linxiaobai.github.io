解压

tar -zvxf redis-2.8.19.tar.gz

安装

打开解压目录
在终端输入 make
然后输入 make test
如果make test如果报错说缺少某个东西的话,你可以试着去用yum install指令安装一下,如果还不型可以跳过这步
然后输入 make install

配置可以后台执行

打开util
输入 ./install_server.sh
然后有提示设置端口号,设置log日志的目录等
然后照着提示一直回车就行,如过你要自定义相关参数和目录的话就输入你要存入的地址就好

运行完成后,如果按默认的设置相关文件目录如下

  • 日志

    /var/log/redis_6379.log
    启动redis服务器的时候,可以在这里看到正常或是异常信息等

  • redis配置文件

    /etc/redis/6379.conf
    启动redis服务器的时候,服务器相关参数在这里配置

  • redis服务启动,关闭,重启

    /etc/init.d/redis_6379

这其实是一个shell脚本,我们其实看一下代码是看懂的

我在这里做了一个小的改动,因为我的redis服务器不希望任何人都能随意访问,所以我设置了密码(也就是在redis.conf里面配置requirepass参数)
但是这时候stop就有问题了,我们中间这段代码,它所谓的关闭服务器是利用了本地的客户端连接服务器,然后执行shutdown命令,如果我们配置了requirepass参数,
那么就得在客户端执行shutdown时加上对于的密码,就是在$CLIEXEC -p $REDISPORT -a $REQUIREPASS shutdown
加上-a $REQUIREPASS

使用

然后我们可以通过ps指令查看当前redis是否启动
输入ps -ef|grep redis
如果看到root 11476 1 0 17:59 ? 00:00:08 /usr/local/bin/redis-server *:6379
说明是启动着的.

我们对应的操作

  • 启动 /etc/init.d/redis_6379 start

  • 关闭 /etc/init.d/redis_6379 stop

  • 重启 /etc/init.d/redis_6379 restart

其实看shell脚本就知道了,start stop和restart对应的是那个$1参数,它用了一个类似于switch的写法执行不同操作罢了.
